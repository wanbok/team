# Example: Web Service Team

A full-stack web application with React/Next.js frontend and Node.js/Python backend.

## Team Composition

| # | Role | Name | Skills |
|---|------|------|--------|
| 1 | PM + Analyst | pm | brainstorming, writing-plans |
| 2 | UI/UX Designer | designer | ui-ux-pro-max |
| 3 | Frontend Dev | frontend-a | test-driven-development |
| 4 | Backend Dev | backend-a | test-driven-development |
| 5 | QA Engineer | qa | verification-before-completion |
| 6 | Oracle | oracle | ask-oracle (Sonnet) |

## File Ownership

| Role | Directories |
|------|-------------|
| frontend-a | `src/`, `public/`, `tests/frontend/` |
| backend-a | `api/`, `db/`, `tests/backend/` |
| designer | `docs/design/` |
| qa | `tests/integration/`, `tests/e2e/` |
| pm | `docs/plans/`, `docs/requirements/` |

## Notes

- For larger projects, add `frontend-b` for a second frontend developer
- Consider adding `backend-b` if there are distinct API domains (auth, billing, etc.)

## Launch Command

```
/team Full-stack web app with React frontend and [backend framework] for [your app description]
```
