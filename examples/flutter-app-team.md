# Example: Flutter App Team

A Flutter mobile app with Dart backend or Firebase.

## Team Composition

| # | Role | Name | Skills |
|---|------|------|--------|
| 1 | PM + Analyst | pm | brainstorming, writing-plans |
| 2 | UI/UX Designer | designer | ui-ux-pro-max |
| 3 | Flutter Dev A | flutter-a | test-driven-development |
| 4 | Flutter Dev B | flutter-b | test-driven-development |
| 5 | QA Engineer | qa | verification-before-completion |

## File Ownership

| Role | Directories |
|------|-------------|
| flutter-a | `lib/features/feature_a/`, `test/features/feature_a/` |
| flutter-b | `lib/features/feature_b/`, `test/features/feature_b/` |
| designer | `docs/design/` |
| qa | `test/integration/`, `integration_test/` |
| pm | `docs/plans/`, `docs/requirements/` |

## Notes

- Oracle is omitted for simplicity; add if needed for complex architecture decisions
- For apps with a custom backend, add `backend-dev` role

## Launch Command

```
/team Flutter mobile app for [your app description]
```
